# Contributing to AI-Assisted Architecture

Thank you for your interest in contributing to AI-Assisted Architecture! This document provides guidelines for contributing to the project.

## Ways to Contribute

### 1. Report Issues
- Bug reports for documentation or scripts
- Suggestions for methodology improvements
- Feature requests for new agents or templates

### 2. Improve Documentation
- Fix typos or clarify explanations
- Add examples or use cases
- Translate documentation

### 3. Contribute Content
- New building blocks (ABBs/SBBs)
- Agent improvements
- Template enhancements
- Tooling guides

### 4. Share Experience
- Case studies from your organization
- Lessons learned
- Best practices

## Contribution Process

### For Minor Changes (typos, clarifications)

1. Fork the repository
2. Make your changes
3. Submit a pull request with a clear description

### For Significant Changes

1. **Open an Issue First**: Describe what you want to contribute
2. **Discuss**: Wait for feedback from maintainers
3. **Fork and Develop**: Create your contribution
4. **Submit PR**: Reference the issue in your pull request

## Pull Request Guidelines

### PR Title Format
```
[TYPE] Brief description

Types:
- [DOCS] Documentation changes
- [METHODOLOGY] Methodology updates
- [AGENT] Agent improvements
- [TEMPLATE] Template changes
- [BB] Building block additions
- [SCRIPT] Script/tooling changes
```

### PR Description Template
```markdown
## Summary
Brief description of changes

## Type of Change
- [ ] Documentation improvement
- [ ] New content (ABB, SBB, pattern)
- [ ] Agent enhancement
- [ ] Bug fix
- [ ] Other (describe)

## Related Issue
Fixes #123

## Checklist
- [ ] I have read the CONTRIBUTING guide
- [ ] Content is organization-agnostic (no company-specific references)
- [ ] Documentation follows the style guide
- [ ] I have tested any scripts locally
```

## Content Guidelines

### Organization-Agnostic Content

All contributions to the core must be:
- **Generic**: No company-specific references
- **Reusable**: Applicable across industries
- **Adaptable**: Easy for organizations to customize

### Writing Style

- Use clear, concise language
- Include practical examples
- Provide rationale for recommendations
- Use consistent terminology (see Glossary)

### Template Contributions

When contributing templates:
1. Include clear instructions
2. Mark required vs optional fields
3. Provide example usage
4. Explain the purpose of each section

### Building Block Contributions

When contributing ABBs or SBBs:
1. Follow the standard schema
2. Include `agent_hints` for AI comprehension
3. Provide traceability links
4. Document integration points

## Code of Conduct

### Our Standards

- Be respectful and inclusive
- Welcome newcomers
- Accept constructive criticism
- Focus on what's best for the community

### Unacceptable Behavior

- Harassment or discrimination
- Trolling or insulting comments
- Publishing others' private information
- Other unprofessional conduct

## Recognition

Contributors will be recognized in:
- CONTRIBUTORS.md file
- Release notes for significant contributions
- Documentation credits where appropriate

## Attribution

When writing about, presenting, or forking this framework,
please credit the original project and link to the repository.
